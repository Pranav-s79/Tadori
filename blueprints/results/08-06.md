one-hop wiring edit to `apps/viz/src/App.tsx`. No `packages/*` touched. No
shared coordination file edited (this result file carries the evidence).

- `deepLink.ts` — `isRootConfined` (pure `..`-segment + absolute/UNC rejection)
  and `buildDeepLink` (per-segment percent-encoding; Windows-drive preserved).
  A non-confined path yields `null`, so the UI renders no link at all.
- `inspectApi.ts` — four fetch wrappers with explicit status discrimination:
  `fetchNodeDetail` (200/404→not_found/409→ambiguous/other→error),
  `fetchNodeEvidence`, `fetchSource` (403/404/409 discriminated), `fetchLinkedDoc`
  (single-doc-or-null). Local mirrors of ToolNode/ToolEdge/Evidence/NodeDetail
  (app cannot import `@tadori/*`).
- `useInspectionStore.ts` — `openEntity`/`goBack`/`close` with ONE-level back
  (not a stack): opening C after B (A previous) drops A; re-opening the current
  entity is a no-op.
- `EvidenceList.tsx` — anchors with confined-only deep links; explicit "+N more"
  only when `evidenceOmittedCount > 0` (never "+0 more").
- `SourceView.tsx` — body shown ONLY when `status==="ok" && staleReason===
  "matches_snapshot" && body!==null`; every other state (incl. a malformed
  ok+non-null-body+stale-reason) renders a suppression notice, never the body.
- `NodeView.tsx` / `EdgeView.tsx` — node header/evidence/source/ADR-body; edge
  provenance triple (origin/confidence/resolution) from non-optional props, so
  an edge cannot render without all three badges. ADR section renders the exact
  string `No documented design decision found.` when no single doc resolves.
- `InspectionPanel.tsx` — single `role="dialog"` root; opening a new entity
  REPLACES content (asserted: exactly one dialog after a second open); close
  button + `Escape`; focus moves into the panel on open; edge/src/dst pivots.
- `App.tsx` — mounts one `InspectionPanel`; search results now open it
  (`SearchPanel openInspectionPanel={…}` — the seam 08-05 stubbed).
