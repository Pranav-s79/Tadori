## What changed (this slice)

Extends the already-existing `GET /api/v1/review/diff` (snapshot↔snapshot,
built by 07-01) with the honesty-critical pagination and coalesce behavior the
09-01 completion-proof requires, as a self-contained, dependency-free increment.

- `packages/server/src/reviewDiffAssembly.ts` (new) — pure functions:
  `parseReviewCursor` / `parseReviewLimit` (validated cursor/limit; invalid →
  null → 400) and `paginateReviewDiff`, which paginates over the combined
  (addedNodes ++ removedNodes ++ edges) sequence with a single offset cursor and
  reports **per-list omitted counts** plus a `nextCursor`. A multi-page fetch
  reconstructs the whole diff with zero duplicate or missing rows.
- `packages/server/src/routes/review.ts` — wired pagination in; added the
  `coalesce=coalesced → 501 coalesced_unsupported` guard (rename/move coalescing
  is 09-02; it must 501 explicitly, never silently return the raw diff labeled
  coalesced) and a `bad_page` 400 for invalid cursor/limit.
- `packages/server/src/errors.ts` — added `notImplemented` (501) helper.
- `packages/server/src/types.ts` — `ReviewDiffDto` gains
  `nodesAddedOmitted`/`nodesRemovedOmitted`/`edgesOmitted`/`nextCursor`
  (additive).
- Tests: `reviewDiffAssembly.test.ts` (new, 6) + 3 new route cases in the
  existing `review.test.ts` (501 coalesce, 400 bad_page, pagination fields).
